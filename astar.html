A* Pathfinding Package

A package for implementing pathfinding AI in any Unity 2D project.

Description

This is a package of 3 scripts that work together to offer a flexible Unity 2D AI agent for the purpose of pathfinding. The user is expected to use the PathfindingMap.cs script to create a grid map and then apply the PathfindingAgent.cs script to whatever object/NPC the user wants to use the A* Pathfinding algorithm.

Details

Engine: Unity (2D)
Programming Language: C#
Role: Programmer
Available to download on Github


The Process

Unity 3D has a component built in with the engine called a Navmesh Agent. If the developer creates a Navmesh for their 3D scene, they can attach a Navmesh Agent component to an object to allow the object to use pathfinding within the scene. There is no official alternative in Unity 2D. Other developers have created packages to solve this lack of a pathfinding agent, and I have also developed a package for personal use.

I started by finding a source I had used for a class to get a full rundown of the A* Pathfinding Algorithm (located here). This method does require some setup, that being that it needs a grid map to function, so I started by creating that, then implementing the above algorithm to work in Unity 2D. One problem I ran into is that it takes a lot of processing power to find the shortest path every machine cycle, so I added a buffer that the user can edit to make it less strenuous. Finally, I added several additional features that may be useful in future projects, such as a debug mode, stopping distance, and object speed.
