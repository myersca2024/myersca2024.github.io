EXT File System

An EXT File System built in a Linux VM using C.

Description

This EXT File System was developed as a final project for my Computer Systems class in my second year. The file system is entirely runnable in Linux to create/delete both files and directories. This is done through creating inodes which store metadata for files and directories, specifically paths and pointers to where they are located in memory.

Details

Programming Language: C
Role: Programmer
Source code available upon request.

Contributions

  • Created system calls for adding and removing directories
  • Developed hard and soft links between inodes and data in memory
  • Added additional functionality of storing the time a file or directory was created

The Process

The final project was split between two assignments. The first part was developing a way to create only files, and the second part would add the additional functionality of directories, links, and timestamping.

The first part was done individually where we used starter code provided by the professor, which was an outline of some of the functions we would need, and were expected to only fill out enough to simply create and delete files in a base directory. We would refer to lectures and other online presentations to figure out how to implement this.

The second part of the project was done in teams where we would use the first part from one of the two team members. We worked collaboratively to develop the rest of the functionality. This included creating and deleting directories, reading/writing to files, creating hard links and symlinks, as well as timestamping inodes (metadata).
